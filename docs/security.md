# 安全设计

## 三层纵深防御

### 第一层：边界防护

| 防护 | 机制 | 对应组件 |
|:-----|:-----|:---------|
| 工作区隔离 | 所有文件操作限制在指定工作目录内 | PathValidator |
| 命令白名单 | 只允许预设的命令被执行 | CommandValidator |
| 文件类型限制 | 只允许读取文本文件 | read_file |

### 第二层：操作管控

| 防护 | 机制 | 对应组件 |
|:-----|:-----|:---------|
| 路径穿越防护 | 用 Path.resolve() 规范化，拒绝 `..` 路径 | PathValidator |
| 命令注入防护 | 全程 shell=False，参数检测危险字符 | CommandValidator |
| SQL 只读校验 | 正则检测 + 关键字拦截，拒绝写入操作 | SQLValidator |
| 输出大小限制 | 最大 1MB/5000 行 | 各工具 |
| 超时控制 | 命令/查询超时自动终止 | run_command / query |

### 第三层：审计追溯

| 功能 | 实现 |
|:-----|:-----|
| 审计日志 | 每次工具调用记录（谁/何时/调用了什么/结果） |
| Trace ID | 每个请求唯一 ID，贯穿调用链路 |
| 脱敏记录 | 审计日志不记录完整文件内容 |

## 安全原则

1. **默认拒绝**：写入类工具默认禁用
2. **纵深防御**：三层防线，单层失效不影响整体安全
3. **最小权限**：命令白名单严格执行
